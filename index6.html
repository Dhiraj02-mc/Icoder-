What is Python?

Python is a high-level, interpreted programming language that is easy to read and learn. It's widely used for web development, data science, automation, scripting, and more.

Quick setup

  • Download Python from python.org or check installation with python -V in terminal.
  • Use an editor like VS Code or Thonny to write and run code.
  • To run: open terminal and use python file.py or use the Run button in your editor.

First program — Hello, World!

Create a new file named hello.py and write:

# hello.py

print("Hello, World!")

Run in terminal: python hello.py — output: Hello, World!

Variables and data types

In Python, you assign a variable using =. No explicit declaration needed.

# examples

x = 5          # int pi = 3.14      # float name = "Dhiraj"  # str (string) is_happy = True   # bool (True/False)

Basic operators

# arithmetic operators

add = 2 + 3 sub = 5 - 1 mul = 4 * 3 div = 10 / 2  # float division floor = 10 // 3 mod = 10 % 3 power = 2 ** 3

Input and output

# user_input.py

name = input("Enter your name: ") print("Hello, " + name + "! Welcome to iCoder.")

Note: input() returns a string. Convert to int() or float() for numbers.

age = int(input("How old are you? ")) print("You are", age, "years old")

Comments

# This is a single-line comment

""" This is a multi-line comment (docstring style) """

Complete example

# greet.py

name = input("Enter your name: ") age = int(input("How old are you? ")) print(f"Hello {name}! You are {age} years old.")

Practice exercises

  1. Write a program that adds two numbers and prints the result.
  2. Ask the user for their name and age, then greet them.
  3. Calculate BMI: ask for weight (kg) and height (m), then print BMI = weight / height**2.
  4. Small quiz: ask "What is 5 + 7?" — if the user answers correctly print "Correct" else print "Incorrect".

Upload each program to iCoder and run to check the results.

Quick tips

  • Choose clear variable names — age, user_name.
  • Run your code frequently — test small parts first.
  • Errors will happen — read error messages carefully and search for solutions.
  • Write comments to explain your code for future you.
